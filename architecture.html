The Integration Pipeline

The full undoRT pipeline moves from Structured Text source to deterministic hardware execution in four stages. Only the first stage — the st2cpp compiler — is currently released. The remaining components are in active development.

The Deterministic Execution Loop

Traditional SoftPLCs introduce overhead through intermediate virtual machines or interpretation layers. undoPLC will bypass this by executing native C++ binaries compiled directly from Structured Text by st2cpp — no VM, no interpreter.

When paired with undoOS (a Linux Preempt-RT kernel configuration), real-time tasks will be pinned to isolated CPU cores using FIFO scheduling priorities. Fieldbus communication via undoBUS (IgH EtherCAT Master) will run deterministically in user-space alongside the PLC runtime.

Today, the generated C++ from st2cpp can already be compiled and run standalone on any C++17-capable platform — the RT integration is the next step, not a prerequisite for using the compiler.

Legacy Migration via undoImport

undoImport will ingest existing commercial IDE configurations — TwinCAT (.tmc/.tpy) and CODESYS projects — extracting POU logic and variable declarations, then routing them directly into the st2cpp pipeline.

This makes undoRT immediately useful for teams with existing automation code, without requiring a full manual rewrite.
