Ecosystem Architecture

Discover how undoRT achieves microsecond-level determinism by integrating custom hardware-level runtime environments with high-level compilation tooling.

The Deterministic Execution Loop

Traditional SoftPLCs introduce overhead through intermediate virtual machines or heavy interpretation layers. undoPLC bypasses this completely by executing native, optimized C++ binaries compiled directly from IEC 61131-3 Structured Text via st2cpp.

When paired with undoOS (Linux Preempt-RT kernel configuration), context switches are minimized. Real-time tasks and fieldbus management via undoBUS (IgH EtherCAT Master) are bound to isolated CPU cores using strict FIFO real-time scheduling priorities, eliminating jitter and non-deterministic latencies.

The Integration Pipeline

The migration suite undoImport allows users to ingest existing commercial IDE configurations (such as TwinCAT or CODESYS projects), extracting the core logical workflows and seamlessly routing them into the open-source undoRT build process.
